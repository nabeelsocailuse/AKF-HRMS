User Management

Solution Name: XpertERP Client: Alkhidmat Foundation Pakistan (AKFP)

Overview

This chapter outlines the principles and processes for managing user accounts and permissions within XpertERP for Alkhidmat Foundation Pakistan. Effective User Management is crucial to ensure data security, maintain confidentiality, and ensure that every user has appropriate access to only the modules and information required for their role. This chapter is primarily intended for System Administrators and authorized IT/HR personnel responsible for managing user access.

Important Note: User Management is a highly sensitive area. Only authorized personnel with explicit approval should create, modify, or deactivate user accounts and roles. Strict adherence to AKFP's IT Security and Data Access policies is mandatory.

Key Concepts in User Management

Before delving into the processes, understanding these core concepts is vital:

  • User: An individual account created in XpertERP, identified by a unique Username and Password, enabling them to log in to the system.
  • Role: A collection of permissions that define what a user can see, create, modify, submit, or cancel within XpertERP. Roles are assigned to users. Examples include "HR User," "Warehouse Officer," "Accountant," "Project Manager," "System Administrator."
  • Permission: A specific authorization to perform an action on a particular document type or module (e.g., "Read" access to Employee records, "Write" access to Stock Entries, "Submit" a Journal Entry). Permissions are granted to Roles.
  • Role Profile: A grouping of multiple roles that can be easily assigned to a user who performs various functions.

Managing User Accounts

This section details the process for creating, modifying, and deactivating user accounts.

Creating a New User Account

When a new employee or authorized personnel requires access to XpertERP, a user account must be created.

  • Process:
    1. Navigate to Home > Users > User.
    2. Click "+ Add User".
    3. Enter the user's "Email Id" (this will be their Username for login).
    4. Enter the user's "Full Name".
    5. Set a "New Password" and confirm it (it's recommended to have the user change this on first login).
    6. Ensure "Enabled" checkbox is ticked.
    7. Assign Roles: In the "Roles" section, click "+ Add Row" and select the appropriate Role(s) for the user (e.g., "HR User," "Accounts User"). Assign roles based on their job function and access requirements as per AKFP's security policy.
    8. (Optional) Link to an existing "Employee" record in the system if the user is an AKFP employee.
    9. Click "Save" then "Submit".
    • Tip: Inform the new user of their login credentials and guide them on initial login and password change.

Modifying an Existing User Account

User account details or roles may need to be updated.

  • Process:
    1. Navigate to Home > Users > User.
    2. Search for and open the specific "User" account.
    3. Click "Edit".
    4. Make necessary changes (e.g., update Full Name, change Password, add/remove Roles).
    5. Click "Save".
    • Tip: When removing roles, ensure the user will still have necessary access for their remaining responsibilities.

Video: Short video showing how to search for a user, open their record, and add/remove a role, then save.

Deactivating / Disabling a User Account

When an employee leaves AKFP or no longer requires system access, their account must be disabled.

  • Process:
    1. Navigate to Home > Users > User.
    2. Search for and open the specific "User" account.
    3. Click "Edit".
    4. Uncheck the "Enabled" checkbox.
    5. Click "Save".
    • Note: Deactivating an account revokes login access but preserves historical data associated with that user. Do not delete user accounts unless specifically instructed and after ensuring all related data integrity is maintained.

Managing Roles and Permissions

This section describes how to define and manage what users can do in the system by configuring roles.

Viewing and Understanding Roles

  • Process:
    1. Navigate to Home > Users > Role.
    2. This list shows all defined roles in XpertERP.
    3. Click on any "Role" (e.g., "Accounts Manager," "Warehouse Officer") to view the specific permissions assigned to it.
  • Understanding Permissions: Within a Role document, you'll see a table listing various document types (e.g., "Employee," "Stock Entry," "Purchase Invoice") and checkboxes for permissions like:
    • Read: Can view the document.
    • Write: Can modify an existing document.
    • Create: Can create a new document.
    • Delete: Can delete a document (rarely granted for critical documents).
    • Submit: Can change a document's status to "Submitted" (often makes it read-only and initiates workflows).
    • Cancel: Can cancel a submitted document.

Video: Short video showing navigation to the Role list, opening one role, and briefly highlighting the different permission checkboxes.

Modifying Role Permissions (Admin Only)

Note: Only System Administrators or highly authorized personnel should modify Role Permissions.

  • Process:
    1. Navigate to Home > Users > Role.
    2. Open the specific "Role" you wish to modify.
    3. Click "Edit".
    4. Adjust the checkboxes for each document type as required. For example, if "Warehouse Officer" needs to submit Stock Entries, ensure "Submit" is checked for "Stock Entry."
    5. Click "Save".
    • Caution: Changes to roles affect all users assigned to that role. Always test changes in a non-production environment if possible, and understand the full impact before saving in the live system.
